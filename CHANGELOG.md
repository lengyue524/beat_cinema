# Changelog

本文档基于 `@_bmad-output/planning-artifacts/prd.md`（Beat Cinema v2 PRD）整理，记录 `v1.0.0` 首个稳定版本的能力交付。

## [v1.0.0] - 2026-03-21

### Added

- 一站式核心工作流：`选歌 -> 搜索 -> 下载 -> 配置/校准`，覆盖 Cinema 配置主路径。
- 关卡资源管理能力：CustomLevels 扫描、`info.dat` 解析、难度徽章、状态标记、筛选与排序。
- 视频搜索与下载能力：支持 YouTube/Bilibili 搜索、URL 直下、下载队列与进度反馈。
- 应用内播放能力：搜索结果支持应用内直接播放，并提供播放失败后的下载兜底路径。
- Playlist 管理能力：`.bplist` 浏览、歌曲映射、缺失歌曲下载、导出与缺失清单提示。
- 媒体增强能力：音乐试听、视频预览、迷你播放器（含进度条与拖拽定位）。

### Changed

- UI 结构升级为 Rail 导航 + 右侧面板工作台，页面切换状态保持。
- 搜索结果交互优化为 `播放 / 网页 / 下载` 三按钮紧凑布局，提升密度与可操作性。
- 设置页精简，移除搜索引擎切换项，降低配置复杂度。
- 歌曲列表项支持标题文本选择复制，提升信息复用效率。

### Performance

- 引入后台解析与缓存策略（含 Isolate 场景），优化大规模关卡列表加载体验。
- 增强 Playlist 哈希索引缓存与重建流程，降低重复解析成本并提供重建入口。

### Fixed

- 修复搜索链路代理透传问题（YouTube/Bilibili 搜索与播放地址解析可使用代理配置）。
- 修复在线视频播放失败时对话框体验问题，失败场景可提示并引导切换下载播放。
- 修复 `yt-dlp` 常见格式/签名失败场景，增加兼容参数与重试兜底。
- 修复多下载任务下的并发修改、刷新风暴、状态更新不一致等稳定性问题。

### Reliability

- 建立日志基础设施：日志落盘、前一天日志自动压缩（zip）、总量 10MB 治理。
- 完善错误码与可观测日志字段，便于播放/下载故障定位与统计分析。
- 加强容错与用户提示，网络/外部工具异常下保持离线能力与可恢复路径。

### Internationalization & Docs

- 补充中英文文案键值，覆盖加载、播放、失败提示、回退确认等场景。
- 新增代理验证手册（中英双语）并在 README 中建立索引入口。
- README/README_CN 功能列表同步补充“应用内播放”等核心能力说明。

### PRD Coverage (v1.0.0)

- 已覆盖（核心）：`FR1-FR7`、`FR8-FR19`、`FR21-FR28`、`FR33-FR41`。
- 持续演进：`FR29-FR32`（首次引导/路径自动检测/更新检测）已具备基础能力，后续继续增强体验与自动化程度。
- 未来规划：Phase 3 Expansion（如 BeatSaver 元数据深度集成、自动偏移检测、跨平台扩展）。
