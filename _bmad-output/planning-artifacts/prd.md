# Product Requirements Document - Beat Cinema v2

**Author:** Lihang
**Date:** 2026-03-10
**Status:** Complete

## 概述

Beat Cinema 是一款 Windows 桌面应用，面向 Beat Saber 自定义关卡玩家。它将关卡浏览、视频搜索（YouTube/Bilibili）、视频下载、音视频同步校准和播放列表管理整合为一站式工作台，解决 Cinema mod 配置流程碎片化、操作繁琐的痛点。

v2 版本将从当前的基础搜索+下载工具升级为完整的关卡资源管理平台。核心改进包括：重新设计的 Spotify 风格布局（Rail 导航 + 右侧按需面板 + 右键菜单）、从 info.dat 解析的带颜色难度徽章的增强列表、内置音乐试听和视频预览、左右声道分离的偏移同步校准工作台、以及 .bplist 播放列表管理与导出。

目标用户为拥有大量自定义关卡（数百首）的 Beat Saber 活跃玩家，尤其是使用 Cinema mod 的核心社区成员。产品创建者即核心用户，所有功能决策均来源于真实使用场景。

### 核心差异化

Beat Saber 社区工具生态中，ModAssistant 管理 mod 安装，BeatSaver/BeastSaber 管理歌曲下载，PlaylistManager 管理播放列表，但没有任何工具覆盖"为关卡配置 Cinema 视频"这一完整流程。Beat Cinema 填补了这个空白，提供从发现视频到同步校准到配置写入的端到端体验。

核心差异化在于工作流的连贯性：选歌 → 搜索 → 下载 → 校准 → 保存，全程不离开一个界面。

## 项目分类

- **项目类型:** Desktop App（Flutter Windows 桌面应用）
- **领域:** Gaming/Entertainment（Beat Saber 工具生态）
- **复杂度:** Medium（媒体播放集成、外部进程调用、文件系统操作）
- **项目上下文:** Brownfield（已有 v0.0.3 代码库，基于 BLoC + GoRouter 架构）

## 成功标准

### 用户成功

- 用户能在**单一界面内**完成从浏览歌曲到配置视频的全流程，无需切换到外部工具或手动编辑 JSON
- 拥有 500+ 自定义关卡的用户能在 **3 秒内** 完成列表加载和首次渲染
- 列表筛选（难度/关键词/有无视频）响应时间 **< 100ms**
- 偏移同步校准通过左右声道分离 + 滑块拖拽，用户能在 **30 秒内** 找到满意的同步点（对比当前手动编辑 JSON 反复试错的分钟级操作）
- 一键下载流程：选歌 → 搜索 → 下载 → 写入配置，**全程不超过 3 次点击**

### 商业成功

- **Phase 1（自用满意）:** 创建者日常使用 Beat Cinema v2 管理所有自定义关卡资源，完全替代手动流程
- **Phase 2（社区推广）:** GitHub 发布后获得 Beat Saber 社区关注，达到 **50+ Stars** 和 **100+ 下载量**
- **Phase 3（社区活跃）:** 收到社区反馈和 Issue 报告，有用户参与功能讨论或贡献

### 技术成功

- **媒体播放稳定性:** media_kit 集成后，音视频播放在 Windows 10/11 上零崩溃
- **文件系统性能:** info.dat 批量解析 + 缓存后，800 首歌的场景下首次加载 < 5s，后续加载 < 1s
- **架构健壮性:** 右侧面板 + IndexedStack 改造后，内存占用增幅 < 30%
- **下载可靠性:** yt-dlp 下载任务成功率 > 95%（排除网络/资源不可用因素）

## 用户旅程

### 旅程 1：日常视频配置（成功路径）

**角色：** 小李，Beat Saber 深度玩家，600+ 自定义关卡，已安装 Cinema mod。

小李刚从 BeatSaver 批量下载了 20 首新歌。他打开 Beat Cinema v2，列表 2 秒内加载完成，新歌自动出现在顶部。每首歌旁显示 info.dat 解析的难度徽章（绿/黄/橙/红/紫），已配置视频的歌曲带视频图标。他右键「Crystallized」→「搜索视频」，右侧面板滑出搜索结果。选中 MV 点击下载，30 秒后状态自动更新为"已配置"。他点击试听，左声道音乐、右声道视频音频——不同步。拖动偏移滑块，5 秒找到同步点，保存。20 分钟配好 8 首歌，切换 Playlist 页面导出「今晚打」歌单，戴上头显开始游戏。

> **能力需求：** FR1-FR7, FR8-FR16, FR17-FR19, FR21-FR24

### 旅程 2：首次使用（引导路径）

**角色：** 阿明，刚装 Cinema mod 的新玩家，150 首关卡，技术能力一般。

阿明从 GitHub Release 下载 Beat Cinema，首次启动检测到路径未设置，引导弹窗出现。他选择 Steam 目录下的 Beat Saber 文件夹，应用验证路径有效。150 首歌加载完成，全部"无视频"状态。他右键一首歌搜索视频，结果带缩略图和时长。选中下载，自动完成配置写入。一首日文歌搜不到结果，他从 YouTube 复制 URL 粘贴到搜索框直接下载。10 分钟配好 5 首歌。

> **能力需求：** FR29-FR32, FR1-FR3, FR8-FR10, FR14

### 旅程 3：批量管理与边缘情况

**角色：** 小李，整理 600 首歌的视频配置状态。

他用「无视频」筛选，列表缩减为 200 首，按难度排序优先处理 Expert+。多选 10 首同作者歌曲批量搜索。遇到异常 info.dat 格式——显示警告但不崩溃；下载遇地区限制——提示失败可重试换源；写入被 Beat Saber 锁定的文件——提示关闭后重试。一小时完成所有 Expert+ 配置，零崩溃。

> **能力需求：** FR4-FR6, FR34-FR37, FR12-FR13

### 旅程 4：Playlist 管理、补全下载与导出

**角色：** 小李，为周末聚会准备精选歌单并打包到另一台设备。

小李切换到 Playlist 页面（页面状态保持），打开「周末精选」30 首歌。系统基于 playlist 歌曲信息映射本地关卡元信息：已下载歌曲使用与“全部歌曲列表”一致的列表项展示；未下载歌曲显示基础信息（来自 playlist）并标记可下载。  
他先点击某首未下载歌曲的“下载”按钮，任务进入下载管理并显示进度；随后点击“下载全部缺失歌曲”，批量补齐剩余缺失曲目。下载过程中个别歌曲失败，系统给出失败原因与重试入口，不阻断其余任务。  
补齐后他点击导出，选择目标目录。系统同时导出 `.bplist` 文件与已下载歌曲目录；若仍有缺失歌曲，导出结果会附带缺失提示清单。最终拷贝到 U 盘，聚会电脑可直接使用已导出的内容。

> **能力需求：** FR17, FR25-FR28, FR38-FR41

## Desktop App 技术需求

### 平台支持

**当前目标：** Windows 10/11（x64）

**跨平台抽象设计原则：**

- 文件路径操作统一使用 `path` 包，禁止硬编码 `\` 分隔符
- Beat Saber 安装路径检测逻辑抽象为平台策略接口（Windows: Registry/Steam 默认路径，未来 Linux: Steam Proton 路径）
- 外部进程调用（yt-dlp）封装为独立服务层，隔离平台差异（可执行文件名、编码处理）
- Process.run 输出编码处理抽象化（Windows GBK vs Linux/Mac UTF-8）
- UI 层使用 Flutter 跨平台组件，避免 Windows 特有 API

### 系统集成

**Beat Saber 文件系统：**

- CustomLevels 目录扫描（递归读取 info.dat）
- info.dat（大小写不敏感）解析，容错处理旧版格式
- cinema-video.json 读写（Cinema mod 配置）
- .bplist 播放列表文件解析与关联

**外部进程：**

- yt-dlp 进程管理（搜索、下载、进度解析）
- yt-dlp 版本检测与兼容性验证
- 进程超时控制、非零退出码处理、输出编码适配

**媒体播放（Growth）：**

- media_kit 音频播放（本地 .ogg/.egg 文件）
- media_kit 视频预览（已下载的视频文件）
- 左右声道分离输出（同步校准工作台）

### 自动更新

- 启动时后台检查 GitHub Releases API（`GET /repos/{owner}/{repo}/releases/latest`）
- 对比本地版本号与远程最新 tag，有更新时显示非侵入式通知
- 用户点击后打开 GitHub Release 页面下载（不做静默自动安装）
- 检测频率：每次启动 + 每 24 小时一次；网络不可用时静默跳过
- 版本号格式遵循 SemVer（v0.0.3 → v2.0.0）

### 离线能力

**完全离线可用：** 关卡列表浏览/筛选/排序、info.dat 解析、已下载视频预览/试听、Playlist 管理/导出、配置编辑

**需要网络：** 视频搜索、视频下载、更新检测

### 窗口与系统行为

- 窗口最小尺寸约束（确保 Rail + 内容区 + 面板布局不被压碎）
- 窗口尺寸和位置持久化（`window_manager` 或类似包）
- 应用关闭时优雅终止正在运行的 yt-dlp 进程
- 大量文件操作使用 Isolate 避免阻塞 UI 线程
- SharedPreferences 存储用户偏好（Beat Saber 路径、窗口状态、上次检查更新时间）

## 功能需求

### 关卡资源浏览

- FR1: 用户可以浏览 Beat Saber CustomLevels 目录下的所有自定义关卡列表
- FR2: 用户可以查看每首关卡的元数据（歌名、作者、BPM、时长、难度及对应颜色标识）
- FR3: 用户可以查看每首关卡的 Cinema 视频配置状态（无视频 / 已配置 / 下载中）
- FR4: 用户可以按关键词搜索关卡列表
- FR5: 用户可以按难度、视频状态、修改时间等条件筛选关卡列表
- FR6: 用户可以按歌名、作者、BPM、修改时间等字段排序关卡列表
- FR7: 系统可以在后台解析 info.dat 并缓存解析结果，列表加载不阻塞 UI

### 视频搜索与下载

- FR8: 用户可以为选定关卡搜索匹配的视频（YouTube / Bilibili）
- FR9: 用户可以通过粘贴视频 URL 直接触发下载
- FR10: 用户可以从搜索结果中选择视频并一键下载
- FR11: 用户可以查看当前所有下载任务的进度状态
- FR12: 用户可以在下载失败时查看错误原因并重试
- FR13: 系统可以管理并发下载队列（限流、排队、超时控制）

### Cinema 配置管理

- FR14: 用户可以为关卡创建 cinema-video.json 配置（关联已下载视频）
- FR15: 用户可以编辑已有的 cinema-video.json 配置参数
- FR16: 用户可以查看关卡目录中已存在的视频文件信息

### 布局与导航

- FR17: 用户可以通过 NavigationRail 在主要功能页面间切换，且页面状态保持不丢失
- FR18: 用户可以通过右键菜单快速访问关卡的常用操作
- FR19: 用户可以通过按需展开的右侧面板查看详情、搜索结果等上下文信息
- FR20: 用户可以调整窗口大小，且布局自适应（有最小尺寸约束）

### 媒体播放（Growth）

- FR21: 用户可以试听关卡的音乐文件
- FR22: 用户可以预览关卡已下载的视频文件
- FR23: 用户可以通过左右声道分离同时播放音乐和视频音轨，并调整偏移量进行同步校准
- FR24: 用户可以保存校准后的偏移值到 cinema-video.json

### Playlist 管理（Growth）

- FR25: 用户可以浏览本地 `.bplist` 播放列表文件
- FR26: 用户可以查看播放列表歌曲，并基于 playlist 歌曲信息映射对应本地关卡元信息；已下载条目复用“全部歌曲列表”一致的列表界面与交互
- FR27: 用户可以查看播放列表歌曲状态（已就绪 / 待处理），并可在详情中查看待处理拆分（未配置、未下载）
- FR28: 用户可以将播放列表涉及的内容导出到指定文件夹
- FR38: 用户可以对未下载歌曲执行单曲下载（来源为 BeatSaver），下载任务进度同步到下载管理
- FR39: 用户可以执行“下载全部缺失歌曲”，并可选择是否对已存在歌曲执行更新；系统应提供明确更新判定（文件缺失/版本不一致/用户强制更新），并在任务创建后 1 秒内将全部任务加入下载管理且可观测状态（pending/downloading/completed/failed）
- FR40: 导出时系统应复制 playlist 文件（`.bplist`）与已下载歌曲目录到目标文件夹
- FR41: 导出遇到部分缺失歌曲时，系统应继续导出可用内容并给出缺失项提示，不因部分失败中断整体导出；导出完成后必须生成缺失清单（至少含 songName/hash/失败原因）并提供“仅重试失败项”入口
- FR42: 歌曲列表支持多选；多选状态下右键菜单提供“添加到歌单、删除歌曲目录”等批量操作
- FR43: Playlist 详情支持删除条目，且删除时提供“是否同步删除歌曲文件目录”的可选项
- FR44: Playlist 详情支持将歌曲“添加到其他歌单”与“移动到其他歌单”
- FR45: 执行添加/移动到歌单时，系统应弹出歌单选择列表并支持歌单搜索

**Playlist 状态语义（用户可见）**
- 已就绪：已配置且可直接使用
- 待处理：未配置或未下载
- 详情区应展示待处理拆分统计：
  - 未配置：已下载但未完成配置
  - 未下载：playlist 中存在但本地未匹配到关卡

### 系统与设置

- FR29: 用户可以设置 Beat Saber 安装路径
- FR30: 系统可以自动检测 Beat Saber 安装路径（Growth）
- FR31: 系统可以在启动时检查应用更新并通知用户（Growth）
- FR32: 用户可以在首次使用时通过引导流程完成基础配置（Growth）
- FR33: 系统可以持久化用户偏好设置（路径、窗口状态）

### 错误处理与容错

- FR34: 系统可以在 info.dat 格式异常时降级显示（仅歌名），不影响其他功能
- FR35: 系统可以在文件被占用时提示用户并提供重试选项
- FR36: 系统可以在外部视频下载进程异常时捕获错误并向用户展示可理解的错误信息
- FR37: 系统可以在网络不可用时正常运行所有离线功能

## 非功能需求

### 性能

- NFR1: 500 首关卡的列表首次加载（含 info.dat 解析）在 3 秒内完成
- NFR2: 缓存生效时后续加载在 1 秒内完成
- NFR3: 列表筛选和排序操作响应时间 < 100ms
- NFR4: 页面切换（NavigationRail）无可感知延迟（< 16ms，单帧渲染）
- NFR5: info.dat 解析在后台 Isolate 中执行，UI 线程帧率保持 60fps
- NFR6: 应用内存占用在 800 首关卡场景下不超过 500MB

### 可靠性

- NFR7: 应用运行期间不因外部数据格式异常（如 info.dat 或外部工具输出）崩溃
- NFR8: 视频下载任务成功率 > 95%（排除网络/资源不可用因素）
- NFR9: 文件写入操作（cinema-video.json）具有原子性——写入失败不损坏已有文件
- NFR10: 应用关闭时在 3 秒内优雅终止所有视频下载相关子进程
- NFR11: 网络不可用时，所有离线功能正常运行，不产生未处理异常

### 集成

- NFR12: 视频下载能力调用封装为可替换服务层，便于适配版本升级或替换工具
- NFR13: Beat Saber 文件路径操作使用跨平台路径抽象，不硬编码平台分隔符
- NFR14: 外部视频下载调用设置 30 秒搜索超时、10 分钟下载超时
- NFR15: 媒体播放器资源在页面离开或应用关闭时正确释放，无内存泄漏
- NFR16: 单曲下载触发后，任务应在 1 秒内出现在下载管理并可观察到状态变化
- NFR17: “下载全部缺失歌曲”在 100 首缺失场景下应保持队列稳定；单任务失败不得导致全队列中断
- NFR18: 导出流程需支持部分成功；失败项记录文件应在导出完成后 3 秒内可见，且至少包含 songName/hash/失败原因/时间戳；用户触发“仅重试失败项”后 1 秒内应创建对应导出重试任务
- NFR19: Playlist 映射策略需可追踪（hash 优先、名称兜底），冲突与未命中场景应有可解释结果
