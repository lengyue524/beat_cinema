# 10-4 核心流程回归护栏清单

适用范围：Beat Cinema v2 核心链路（搜索下载、配置写入、面板流转）

执行模式：
- `smoke`：每次功能改动后执行，目标 10-15 分钟
- `full`：版本发布前执行，目标 30-45 分钟

---

## 0. 前置条件

- Beat Saber 路径已正确配置
- 至少存在 1 个可下载的 playlist 未安装曲目
- 至少存在 1 个本地已下载但未配置的视频曲目
- 网络可用（用于搜索/下载）

失败判定（通用）：
- 页面无响应超过 3 秒
- 状态变化不符合预期且无可解释提示
- 单任务失败导致其他任务链路中断

证据采集（必填）：
- 执行时间：
- 执行人：
- 目标故事/分支：
- 相关日志位置（下载管理/playlist 列表/错误提示）：
- 结论：`pass/fail`

---

## 1) 回归链路 A：搜索 -> 下载任务 -> 状态反馈

### A1. 单曲下载（smoke 必测）

步骤：
1. 进入 Playlist 详情页，选择一个“未安装”曲目并点击下载。
2. 观察下载管理与 Playlist 行内状态。
3. 等待任务完成。

预期状态变化：
- 1 秒内任务出现在下载管理（`pending/downloading`）
- 行内下载按钮变为进度态（转圈）
- 完成后该条目从“未安装列表”移除或状态转为已匹配
- 未安装计数自动减少

失败反馈要求：
- 下载失败时显示可读错误文案（行内或汇总）
- 失败后停止转圈，允许重试
- 不影响其他曲目继续下载

可判定锚点（必须同时满足）：
- 错误提示可在 Playlist 缺失列表区域或统一错误提示区看到（不可仅控制台日志）
- 失败后该条目不再处于 `downloading=true` 状态
- 重试入口可操作（单曲下载按钮或失败重试动作）

### A2. 批量下载（full 必测）

步骤：
1. 点击“下载全部缺失歌曲”。
2. 观察任务批量入队、并发推进与完成回流。

预期状态变化：
- 1 秒内创建可观察任务集
- 队列稳定推进，失败项不阻断整体
- 任务稳定后未安装计数收敛到最终值

失败反馈要求：
- 对失败项可识别（失败原因可见）
- 能继续执行“仅重试失败项”或单曲重试

可判定锚点（必须同时满足）：
- 失败项在导出/下载结果中可定位（至少含 songName/hash/失败原因）
- 其余成功项状态正常回流，不出现“整体卡住”

---

## 2) 回归链路 B：下载完成 -> 配置写入 -> UI 同步

### B1. 自动配置写入（smoke 必测）

步骤：
1. 在搜索面板对某歌曲执行下载（或 URL 下载）。
2. 下载完成后检查目标关卡目录 `cinema-video.json`。
3. 返回列表观察状态图标和摘要。

预期状态变化：
- 下载成功后自动写入配置文件（原子写入）
- 列表图标由“未配置/下载中”切换为“已配置/已就绪”
- 摘要计数同步更新

失败反馈要求：
- 写入失败时有明确错误提示与重试路径
- 失败不会导致应用崩溃或卡死

可判定锚点（必须同时满足）：
- `cinema-video.json` 内容结构可被现有读取链路正确识别
- 状态图标变化与摘要变化在同一操作周期内可观察

### B2. 异常路径（full 必测）

步骤：
1. 模拟文件锁定或无权限写入（可通过占用文件方式复现）。
2. 重试配置写入。

预期状态变化：
- 展示可解释失败原因
- 重试后可恢复成功
- 状态机不进入“永久下载中”或“无反馈”死状态

可判定锚点（必须同时满足）：
- 失败时 `downloadError` 字段可见或等效错误状态可见
- 失败后可触发重试并得到新状态结果

---

## 3) 回归链路 C：右键入口 -> 面板流转 -> 状态保持

### C1. 面板开关与上下文切换（smoke 必测）

步骤：
1. 在列表右键歌曲 A 打开搜索面板。
2. 不关闭面板，右键歌曲 B 切换上下文。
3. 返回 Playlist 页面并再次进入详情。

预期状态变化：
- 面板可在 300ms 级别打开并展示正确上下文
- 切换目标歌曲后上下文即时更新
- 页面切换后保留合理状态（选中项、可见统计、无异常重置）

失败反馈要求：
- 如果目标上下文缺失，需有安全降级提示（而非空白或崩溃）

可判定锚点（必须同时满足）：
- 上下文切换后目标歌曲信息与当前操作对象一致
- 页面重进后无异常重置（不出现全空、错绑、崩溃）

- 与 PRD 对齐：FR26/FR38/FR39，NFR16/NFR17/NFR19
- 与 Project Context 对齐：BLoC 错误状态传播、路由层级、L10n、禁止 `print`
