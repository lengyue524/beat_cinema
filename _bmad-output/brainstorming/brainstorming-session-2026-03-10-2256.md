## Phase 1: SCAMPER 发散探索

### 核心设计决策

**功能 1 — 布局与导航:**
- **最终方案**: Spotify 变体 — Rail 保持导航切换主内容区 + 右侧面板按需展开 + 右键菜单
- 右侧面板触发: 混合模式（右键菜单操作 + 面板独立开关按钮）
- 面板作用域: 仅音乐列表页面可用，切换到其他页面自动隐藏，回到列表时恢复状态
- 页面保持: IndexedStack 避免切换时重建

**功能 2 — 难度显示:**
- Beat Saber 原生颜色编码徽章（紫色 Expert+、红色 Expert、橙色 Hard 等）

**功能 3 — 音乐试听:**
- 列表 item 内播放按钮（传统方案）

**功能 4 — 偏移同步:**
- 文本输入框 + 可拖拽时间轴滑块，双重控制

**功能 5 — Playlist 导出:**
- 从 .bplist 读取 → 筛选匹配本地歌曲 → 复制到指定目录（资源归集工具）

## Phase 2: 思维导图 — 模式识别

### 4 个关键模式

**模式 A: 共享基础设施**

| 基础设施 | 依赖它的功能 |
|----------|-------------|
| 右侧面板系统 | 搜索结果、视频预览、同步工作台、Dashboard |
| 右键菜单系统 | 搜索视频、添加到 Playlist、批量操作、导出 |
| 媒体播放引擎 | 音乐试听、视频预览、偏移同步（左右声道） |
| info.dat 解析器 | 难度显示、颜色徽章、BPM、时长、歌曲元数据 |

**模式 B: 用户工作流链**

```
发现歌曲 → 筛选难度 → 搜索视频 → 一键下载 → 偏移校准 → 加入 Playlist → 导出分享
```

**模式 C: 渐进式复杂度**

| 层级 | 功能 | 复杂度 |
|------|------|--------|
| L1 基础 | 页面保持、难度显示、状态标记、列头排序 | 低 |
| L2 交互 | 右键菜单、右侧面板、密度切换、多选操作 | 中 |
| L3 媒体 | 音乐试听、视频预览、智能搜索联动、一键下载 | 高 |
| L4 高级 | 偏移同步工作台、自动检测偏移、反向匹配 | 高 |
| L5 管理 | Playlist 管理、难度分组导出、备份工具 | 中 |

**模式 D: 新增依赖**

| 能力 | 需要的包 |
|------|---------|
| 音视频播放 | `media_kit` + 平台 libs |
| 声道分离 | `media_kit` 或 `just_audio` |
| .bplist 解析 | 无需新包（JSON 格式） |
| BeatSaver API | `http` 或 `dio` |
